Tres formas de entrar al mismo proyecto

Cada rol responde pregunta distinta, pero comparte misma fuente de datos y mismo objetivo: entender qué hace fuerte a película popular.

Analista

¿Qué películas arrasan y por qué?

Explora top semanal de TMDb. Busca patrones por género, idioma, mes de estreno y desempeño histórico.

  • Análisis de distribución de ratings por género
  • Meses con más estrenos exitosos
  • Correlación entre presupuesto y recaudación
  • Dashboard en Tableau o Metabase
Ingeniero

Pipeline diario automatizado

Construye flujo que consulta TMDb cada día, limpia registros y deja dataset listo para análisis o modelado.

  • Script Python para llamar API de forma diaria
  • Limpieza de duplicados, nulos y normalización
  • Persistencia en SQLite local o PostgreSQL
  • Automatización con cron o Airflow básico
Científico

Predecir rating antes del estreno

Entrena modelo de regresión con datos disponibles antes del lanzamiento para estimar recepción esperada.

  • Features: género, presupuesto, director y mes
  • Modelos base: Random Forest o XGBoost
  • Evaluación con MAE y $R^2$
  • Mini app en Streamlit para demo

API real, barata y suficiente para proyecto serio

TMDb da catálogo amplio, buen detalle y curva de entrada baja. Sirve para practicar extracción, limpieza, análisis y predicción sin inventar dataset.

The Movie Database (TMDb)

API REST con más de 900 mil películas, ratings, presupuesto, recaudación, géneros, créditos y fechas de estreno. Solo hace falta cuenta y API key.

api.themoviedb.org/3/movie/popular?api_key=TU_KEY&language=es

Endpoints clave

  • /movie/popular para ranking semanal
  • /movie/top_rated para comparación
  • /movie/{id} para detalle y presupuesto
  • /genre/movie/list para catálogo de géneros
  • /discover/movie para filtros avanzados

Campos útiles por película

  • Título, idioma y fecha de estreno
  • Vote average y total de votos
  • Presupuesto y recaudación en USD
  • Géneros para agrupación y features
  • Director y actores principales

Ruta de ejecución, sin humo

Orden correcto: acceso, ingesta, análisis, modelo y entrega. Así proyecto avanza sin atascarse en etapa equivocada.

01

Regístrate en TMDb y genera API key

Entra a themoviedb.org, crea cuenta y genera llave en Settings → API. Paso rápido, pero bloquea todo si no queda listo.

02

Escribe pipeline de ingesta

Consulta /movie/popular y /movie/{id}, limpia respuesta y guarda tablas como movies y genres en SQLite.

03

Explora distribución y patrones

Usa Pandas y plots para responder preguntas concretas: géneros fuertes, meses con blockbusters y señales de éxito.

04

Entrena modelo predictivo

Target: vote_average. Features iniciales: budget, genre_id, release_month y runtime. Evalúa con MAE antes de iterar.

05

Publica demo con Streamlit

Cierra proyecto con mini app donde usuario carga datos de película futura y recibe predicción de rating.

Stack recomendado por rol

Nada exótico. Herramientas estándar, rápidas de explicar y suficientes para producir pipeline, análisis y demo final.

Ingeniero
  • Python + requests para ingesta
  • SQLite o PostgreSQL para persistencia
  • Pandas para limpieza
  • cron o Airflow básico para scheduling

Ideal si quieres mostrar ETL completo y disciplina de datos reproducible.

Analista
  • Pandas + Matplotlib
  • Seaborn para correlaciones
  • Metabase o Tableau
  • SQL para consultas y métricas

Buena mezcla para contar historia con datos sin meter complejidad de más.

Científico
  • Scikit-learn para baseline robusto
  • Random Forest o XGBoost
  • Streamlit para demo final
  • Joblib para guardar modelo

Suficiente para pasar de feature engineering a producto demostrable en poco tiempo.
